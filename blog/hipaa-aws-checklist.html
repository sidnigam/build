HIPAA on AWS: The Architecture Checklist That Passes Audits

By Sid Nigam · February 20, 2026 · 8 min read

We've helped healthcare clients pass HIPAA audits on the first attempt — every time. The secret isn't complexity; it's knowing exactly which AWS controls auditors check and configuring them correctly from day one. Here's the checklist we use.

1. Data Encryption (In Transit & At Rest)

  • All S3 buckets: SSE-KMS with customer-managed keys, bucket policy denying non-HTTPS uploads
  • RDS: encryption at rest enabled at creation (cannot be added later without a snapshot restore)
  • EBS volumes: encrypted at account level via default encryption setting
  • All data in transit: TLS 1.2+ enforced via ALB listeners and security group rules

2. Network Isolation

  • PHI workloads in private subnets only — no public IP assignment
  • Separate VPC for production, with VPC Flow Logs enabled to CloudWatch
  • Security groups following least-privilege: no 0.0.0.0/0 inbound rules
  • AWS PrivateLink for all managed service access (S3, DynamoDB, SSM)

3. Access Control & Identity

  • IAM roles with least-privilege policies — no wildcard (*) actions on PHI resources
  • MFA enforced for all IAM users via SCP at the Organization level
  • AWS SSO (IAM Identity Center) for human access — no long-lived access keys
  • AWS Secrets Manager for application credentials rotation (90-day rotation policy)

4. Audit Logging

  • CloudTrail: multi-region trail, S3 delivery to locked bucket (Object Lock), log validation enabled
  • S3 server access logging on all PHI buckets
  • CloudWatch Logs for application logs with 7-year retention for HIPAA
  • AWS Config with managed rules for continuous compliance checking

5. Incident Response

  • Amazon GuardDuty enabled across all accounts
  • AWS Security Hub with HIPAA standard enabled
  • SNS alerts on GuardDuty High/Critical findings to on-call team
  • Documented breach response runbook with <60 minute detection-to-notification SLA

6. Business Associate Agreement (BAA)

Before using any AWS service with PHI, sign a BAA with AWS. This is done through the AWS Artifact console. Only services covered under the BAA may process PHI — confirm the full list at aws.amazon.com/compliance/hipaa-eligible-services-reference/.
